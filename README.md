## 📖 About Karigar

**Karigar** ("craftsman" / skilled worker) is a proposed platform designed to connect customers with trusted, verified skilled-service providers (electricians, plumbers, mechanics, and similar tradespeople). This repository documents the full requirements engineering and software design phase for the platform — covering everything a development team would need to begin implementation.


## 📦 What's Inside

This project follows a complete Software Engineering documentation pipeline:

- ✅ **Requirements gathering & analysis** — functional and non-functional requirements
- ✅ **Use Case Diagrams** — actor-system interactions across the platform
- ✅ **Data Flow Diagrams (DFDs)** — Level 0/1 system data flow
- ✅ **Entity Relationship Diagrams (ERD)** — data model for the platform
- ✅ **Class Diagrams** — object-oriented structure of the system
- ✅ **Sequence Diagrams** — interaction flow for key use cases (booking a service, verifying a "karigar," etc.)

## 🎯 Why This Project

Karigar addresses a real gap — finding *verified, trustworthy* skilled labor is informal and risky in most local markets. This project's design phase focuses heavily on the **trust and verification layer** (ratings, service history, identity verification flows) as a core differentiator, modeled out fully in the use case and sequence diagrams before any code was written.
